## 评估策略

**实时验证**：生成代码片段 → 立即运行测试 → 通过 → 下一步

| 验证方式 | 阻塞？ | 说明 |
|----------|--------|------|
| 单元测试(pytest) | ✅ 阻塞 | 核心逻辑必须通过 |
| 类型检查(mypy) | ✅ 阻塞 | 不变量I-5强制 |
| 静态分析(ruff) | ⚠️ 仅警告 | 风格问题不阻塞 |
| 运行时测试 | ✅ 阻塞 | 关键路径 |

---

## 自动修复

最多3次自动修复尝试。超过则标记失败+记录原因。
