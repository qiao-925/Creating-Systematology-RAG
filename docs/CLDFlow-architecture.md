## 五层职责边界

### 输入层

**职责**：接收查询、增强检索、标准化输入

| 子模块 | 职责 | 输出 |
|--------|------|------|
| 查询增强 | HyDE + 多查询(3-5角度) | 增强后查询列表 |
| 文档检索 | 按数据源分级检索 | 文档集合 |
| 饱和度检测 | 重复>70%或10轮/5查询硬限制 | 停止信号 |
| 来源验证 | T1-T4分级 | 来源层级标记 |

**输出契约**：`ParsedQuery`（query_text + documents + context）

---

### CLD层（因果结构提取）

**职责**：从文档中提取因果变量和关系，构建共享因果图

| 子模块 | 职责 | 输出 |
|--------|------|------|
| 动态视角生成 | Conductor 根据问题生成3-5个Specialist角色 | Perspective[] |
| Specialist Agent | 按视角提取因果链（JSON Schema输出） | CausalLink[] |
| 节点归并器 | 余弦相似度>0.8自动归并 | NodeCluster[] |
| 冲突检测器 | 计算分歧度，分级处理 | 低/中/高分歧标记 |
| 裁判Agent | 高分歧(>0.5)元投票评审 | 融合后的CausalLink |
| CLD构建器 | 组装共享因果图 | NetworkX DiGraph |

**全自动流程**：
```
Conductor 分析问题 → 生成视角
    ↓
N个Specialist并行提取（JSON Schema输出）
    ↓
节点归并（阈值0.8自动处理）
    ↓
冲突检测（计算分歧度）
    ├── 低分歧(<0.3) → 加权平均自动聚合
    ├── 中分歧(0.3-0.5) → 加权平均+置信度标记
    └── 高分歧(>0.5) → 送裁判Agent
    ↓
自审（不变量I-7：结构合理性检查）
    ↓
SharedCLD (NetworkX DiGraph)
```

**输出契约**：`SharedCLD`（nodes + edges + metadata）

---

### FCM层（模糊认知图）

**职责**：将定性因果图转化为定量权重矩阵，进行场景仿真

| 子模块 | 职责 | 输出 |
|--------|------|------|
| 权重聚合器 | 多Agent语言权重→数值权重 | 权重矩阵 W[n×n] |
| 仿真引擎 | Kosko迭代求稳态 | 稳态状态向量 |
| 场景对比器 | 基准场景 vs 干预场景 | 差异矩阵 |

**全自动流程**：
```
SharedCLD输入
    ↓
每条边：各视角语言评级（±L/M/H/VH）
    ↓
7档映射 → 数值权重（±0.3/0.5/0.7/0.9）
    ↓
均值聚合 → 权重矩阵
    ↓
Kosko仿真（Tanh激活，100轮或收敛）
    ├── 基准场景（当前状态）
    └── 干预场景（政策干预）
    ↓
自审（收敛性检查）
    ↓
权重矩阵 + 场景对比结果
```

**输出契约**：`WeightedFCM`（weight_matrix + confidence_matrix + baseline_state + intervention_states）

---

### D2D层（杠杆点分析）

**职责**：识别高影响力节点，排序政策干预优先级

| 子模块 | 职责 | 输出 |
|--------|------|------|
| 敏感性分析器 | 单节点10%扰动，测系统响应 | 影响力分数 |
| 不确定性计算器 | 多视角权重分歧→置信度区间 | 置信度标记 |
| 杠杆点排序器 | 按影响力+置信度综合排序 | 杠杆点列表 |

**全自动流程**：
```
权重矩阵输入
    ↓
对每个节点：
    ├── 施加+10%扰动
    ├── 重新仿真
    └── 计算||新稳态-原稳态||
    ↓
按影响力排序
    ↓
叠加置信度标记（来自权重分歧）
    ↓
自审（排序合理性检查）
    ↓
杠杆点排序（高影响力+高/中/低置信度）
```

**输出契约**：`LeverageAnalysis`（leverage_points + uncertainty_ranges）

---

### 输出层

**职责**：结构化呈现分析结果

1. 因果结构图（CLD可视化）
2. 场景对比表（FCM仿真结果）
3. 杠杆点排序（影响力+置信度）
4. 证据追溯（来源Agent+原文引用+来源层级）

## 层间接口契约

### 接口1：输入层 → CLD层

```yaml
输入: ParsedQuery
├── query_text: str
├── documents: List[Document]
└── context: Dict

输出: CausalLink[] (JSON Schema, 不变量I-2强制)
├── source: str
├── target: str
├── polarity: Literal["+", "-"]
├── evidence: str
└── extracted_by: str
```

### 接口2：CLD层 → FCM层

```yaml
输入: SharedCLD (NetworkX DiGraph)
├── 节点列表（归并后的变量）
└── 边列表（融合的CausalLink）

输出: WeightedFCM
├── weight_matrix: float[n][n]
├── confidence_matrix: float[n][n]
└── divergence_notes: List[str]
```

### 接口3：FCM层 → D2D层

```yaml
输入: WeightedFCM
├── weight_matrix
├── baseline_state: float[n]
└── intervention_states: Dict[str, float[n]]

输出: LeverageAnalysis
├── leverage_points: List[NodeImpact]
│   ├── node: str
│   ├── impact_score: float
│   ├── confidence: Literal["high", "medium", "low"]
│   └── affected_nodes: List[str]
└── uncertainty_ranges: Dict[str, Tuple[float, float]]
```

### 接口4：D2D层 → 输出层

```yaml
输入: LeverageAnalysis + SharedCLD + FCMResults

输出: StructuredReport
├── cld_visualization: GraphViz
├── scenario_comparison: Table
├── leverage_ranking: List[Recommendation]
└── evidence_tracing: Dict[str, Citation]
```

## 冲突检测与消解

### 分歧度计算

```python
disagreement = 1 - abs(plus_count - minus_count) / total
# 3:0 → 0（无分歧）
# 3:1 → 0.5（中等分歧）
# 2:2 → 1.0（完全分歧）
```

### 决策矩阵

| 分歧度 | 处理方式 | 置信度标记 |
|--------|----------|-----------|
| 0 | 直接采纳 | high |
| <0.3 | 多数投票 | high |
| 0.3-0.5 | 加权平均+标记 | medium |
| >0.5 | 裁判Agent评审 | low |

> 详细冲突消解设计见 `docs/cldflow/cld-conflict-resolution.md`

## 动态视角生成

Conductor 根据问题特征动态生成3-5个Specialist Agent：

| 问题复杂度 | 视角数量 | 示例 |
|-----------|---------|------|
| 简单 | 2-3 | 财政 + 市场 + 社会 |
| 中等 | 3-4 | 法律 + 财政 + 市场 + 社会 |
| 复杂 | 4-5 | 政策 + 技术 + 经济 + 环境 + 社会 |

Phase 1：预定义模板 + LLM优化（混合方案）
Phase 2：完全动态生成

> 详细设计见 `docs/cldflow/dynamic-agent.md`
