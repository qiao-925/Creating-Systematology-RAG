# 从综合集成法案例到 Agent 架构洞察

> 2026-04-10 对话记录。状态：**思维路径已梳理，待系统工程方法论落地**。

## 起点：寻找方法论的具体案例

在 `brainstorm-product-direction` 中确认了"以系统科学为方法论内核"的方向后，需要一个**真实的应用案例**来验证方法论如何落地。

回忆起读过的一篇关于"从定性到定量综合集成法"在工资和物价研究中的应用。经查证，这是：

> **「关于财政补贴、价格、工资的综合研究」**
> 马宾、于景元等人，1980 年代中期。
> 钱学森从此案例中提炼出"从定性到定量综合集成法"。
> 既是"开放的复杂巨系统"概念提炼的重要实践基础，更是方法论提炼的主要依据。

## 第一步：案例方法论结构还原

从文献描述中还原了案例的研究过程：

| 步骤 | 做了什么 |
|------|---------|
| 问题界定 | 财政补贴负担沉重、物价与工资联动复杂、多方利益冲突 |
| 多视角定性分析 | 经济学家看价格机制、管理专家看财政约束、系统工程专家看子系统耦合 |
| 问题结构化 | 识别关键变量和关系：补贴额度↔物价水平↔工资调整↔财政赤字↔居民实际收入 |
| 建模与方案生成 | 不同补贴削减方案下的物价变动、工资补偿需求 → 多个方案及量化后果 |
| 综合集成 | 把定量结果拿回给各方专家，判断哪个方案在政治、经济、社会维度上最可行 |
| 迭代 | 第一轮结果暴露新问题，回到定性分析 |

**关键特征**：不是"搜资料 → 写总结"，而是"多视角定性 → 结构化为可计算关系 → 量化比较多方案 → 综合判断"。

## 第二步：多视角 → 多 Agent

### 问题：视角纯度

LLM 在单次调用中很难做到"只从一个视角思考"——它天然会综合。让一个 Agent 扮演经济学家，它会忍不住加入系统视角，导致视角不纯。

### 结论：用多 Agent 保证视角隔离

每个 Perspective Agent 通过三层保证纯度：
1. **Prompt 隔离**：system prompt 只定义一个分析框架
2. **知识隔离**：RAG 检索范围限定在自己领域
3. **工具隔离**：通过 MCP 集成领域专属工具，限制 Agent 的能力边界

工具这一层很重要——光靠 prompt 定义专业度是软约束，**工具决定了 Agent 能做什么计算和操作**，是硬约束。

## 第三步：自我批评 → 发现缺失的核心

### 第一版结构（被否定）

画了一个标准的多 Agent 结构：多个 Perspective Agent → Synthesizer → 输出。

**问题**：这是通用多 Agent 模板（CrewAI/AutoGen 都能画出来），和综合集成法没有本质关系。换个标签就是任何领域的 Agent Team。

### 关键发现：模型在哪里？

综合集成法的核心不是"多专家讨论"——那只是研讨会。它的核心创新是：

```
专家的定性认识 ──→ 被形式化为可计算的模型 ──→ 模型跑出定量预测 ──→ 预测反馈回专家修正认识
```

在价格-工资案例里：
1. 经济学家说"取消补贴会引起物价上涨"（定性）
2. 被转化为 `价格变动 = f(补贴削减比例, 市场弹性, 替代品可用度)`（定量模型）
3. 模型预测：削减 50% → 物价涨 15% → 需要工资补偿 Y%（定量）
4. 政策专家看到结果："工资补偿假设不对，因为..."（定性修正）
5. 回到 2，修改模型

**中心是一个不断被修正的共享模型，不是任何一个 Agent。**

## 第四步：共享模型的工程形态 → 因果回路图（CLD）

系统动力学的因果回路图是"共享模型"的最小可行形态：

```
    财政补贴额 ──(-)──→ 物价水平
        ↑                  │
       (+)                (+)
        │                  ↓
    财政赤字 ←──(+)── 工资补偿需求
        │                  ↑
       (+)                (+)
        ↓                  │
    削减补贴压力 ──(+)──→ 物价上涨预期
```

### 映射关系

| 系统动力学概念 | Agent 架构对应物 |
|-------------|----------------|
| 因果回路图 (CLD) | 共享模型的核心数据结构 |
| 存量-流量 | CLD 中节点的类型标注 |
| 反馈环路识别 | 图算法（遍历 CLD 找环路） |
| 杠杆点分析 | Synthesizer 的核心能力——找杠杆，不是总结 |
| 延迟 | CLD 边的属性（fast/slow） |
| 系统基模 | 模式库——已知结构模式 → 已知行为后果 |

### 关键洞察

不需要一开始就做数值仿真。**定性推理在因果图上就能做**——"沿着正反馈环走一圈，判断是加强还是衰减"。这比数值模拟简单得多，但已经比纯文本分析强很多了。

## 当前结论

### 已确认

1. **综合集成法的核心不是多专家讨论，是「共享模型」**——因果回路图作为定性到定量的桥梁
2. **多 Agent 保证视角纯度**，Prompt + Knowledge + Tools (MCP) 三层隔离
3. **收敛的驱动力是模型推演 vs 专家直觉的张力消解**，不是轮次耗尽
4. **CLD 的定性推理是最小可行的「定量」**——不需要数值仿真即可产出结构性洞察

### 修正后的架构概念

```
                    Conductor (编排，不分析)
                       │
          ┌────────────┼────────────┐
          ▼            ▼            ▼
       Agent A      Agent B      Agent C
       (定性判断)    (定性判断)    (定性判断)
       [MCP工具]    [MCP工具]    [MCP工具]
          │            │            │
          └────────────┼────────────┘
                       ▼
              ┌─────────────────┐
              │   共享模型 (CLD) │  ← 架构核心
              │   被判断不断修正  │     不是 Agent，是数据结构
              └────────┬────────┘
                       ▼ 定性推演
              ┌─────────────────┐
              │   推演结果       │
              └────────┬────────┘
                       ▼
              推演 vs 专家直觉 有张力?
                  │              │
                 否              是
                  ▼              ▼
               收敛输出       新一轮定性分析
```
