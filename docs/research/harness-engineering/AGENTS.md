# AGENTS.md

## Stable Rules

- 将 [MVP执行总计划.md](/home/q/Desktop/H&C%20code-workspace/Harness-Engineering-Learning/MVP%20Version/MVP%E6%89%A7%E8%A1%8C%E6%80%BB%E8%AE%A1%E5%88%92.md) 作为 MVP 规划的唯一主文档。核心判断优先在这份文档中做增量更新，不再分散到多份并行主文档中。
- 规划推进按层次进行：方向 -> 架构骨架 -> 职责边界 -> 细节设计 -> 旧系统承接。当前层没有稳定之前，不要提前跳到下一层。
- 这个项目优先采用高信号、强收束的规划方式，而不是宽泛发散。先冻结核心，再一次只展开一个问题。
- 对长文档的开头概括，优先使用这个结构：`定义 + 命题 + 架构 + 路径`。
- 文档组织优先采用渐进式披露：总计划保留宏观判断、冻结结论、推进顺序和文档地图，具体展开下沉到子文档并从主文档引用。
- 在尚未进入实现阶段时，优先保留“最小冻结集”，删除解释型铺垫和过程性展开；子文档降级为备查，不抢主文档注意力。

## Candidate Rules

- 对长文档，优先在开头给出高密度概括，让读者只看开头就能抓住全文核心，再往下展开。
- 对复杂代码文件，可以考虑在文件顶部补一小段文件级概括，说明：这个文件的核心职责是什么、主要由哪几部分组成、不负责什么。

## Notes

- 只有经过验证、重复证明有价值的规则，才进入 `Stable Rules`。
- `Candidate Rules` 作为项目进行中的轻量沉淀区使用，后续定期把其中内容升级、修订或删除。
