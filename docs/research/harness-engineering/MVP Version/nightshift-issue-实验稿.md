# 研究型 Agent MVP 自主探索实验

## Goal

在 6 到 9 小时预算内，把
`docs/research/harness-engineering/MVP Version/研究型Agent-MVP定义与执行计划.md`
作为北极星文档，基于当前仓库现实，自主探索并推进“研究型 Agent MVP”。

这里的目标不是机械执行原文，而是允许你在边界内质疑、重组、修正原计划，
并选择本轮最有价值的推进路径，尽可能产出高价值的设计、实现、验证和文档回写。

## Current Phase

Autonomous Experiment 1

从这份总计划文档与当前仓库出发，自主决定本轮最值得推进的主路径。
允许你在以下几类动作中自行排序和取舍，但必须围绕“研究型 Agent MVP”主命题服务：

1. 收敛研究内核、有限自主边界和最小闭环。
2. 把设计映射到当前仓库，形成首个可运行竖切片方案。
3. 如果置信度足够，在允许范围内直接落下最小原型、骨架或关键实现。
4. 完成必要的验证、回写文档、记录风险和下一轮建议。

你不需要逐条服从原计划文本。
你需要做的是：在预算内，判断什么最值得做，并把它推进到一个像样的程度。

## Done Definition

本轮结束时，至少满足以下之一：

- 形成一组足够支撑下一轮实现的高质量设计工件；
- 在当前仓库中落下一个最小可运行竖切片或原型骨架；
- 同时完成上面两者。

并且必须满足：

1. 明确说明你选择了什么路径，为什么这样选。
2. 明确说明做了什么、没做什么、为什么没做。
3. 明确说明当前风险、已知边界、下一轮建议。
4. 对改动过的内容做最小必要验证。
5. 如果阶段完成并满足条件，则创建 checkpoint。

## Constraints

1. 时间预算 8 小时，允许你自主规划阶段，但不要无止境发散。
2. 允许你质疑和重组原计划，但不要脱离“研究型 Agent MVP”主线。
3. 允许修改 `docs`、`backend`、`frontend`、`tests`、`prompts`、`scripts`、`.agent`、`.github` 等允许目录。
4. 不允许默认引入多 Agent、长期记忆、重型状态机、平台化扩张。
5. 不允许新增依赖、修改 public API、认证逻辑、数据库 schema，除非明确记录并升级给人。
6. 不要为了显得完整而扩写低价值材料。
7. 如果发现现有仓库 baseline 问题阻塞推进，可以做有限止血，但不要把整夜都耗在无关清扫上。

## Review Focus

1. 是否真的推进了“研究型 Agent MVP”，而不是只扩写文档。
2. 是否做出了判断和取舍，而不是材料堆砌。
3. 是否把最有价值的 1 到 2 件事做深，而不是很多事情都只碰一下。
4. 文档、代码、验证之间是否一致。
5. 是否出现了超范围设计或无意义复杂化。
