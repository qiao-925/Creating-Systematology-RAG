# AI 研究工具竞品调研

> 2026-04-10 桌面调研。聚焦"AI 辅助研究"赛道。

---

## 赛道分层

### 第一梯队：学术垂直研究工具

| 产品 | 核心功能 | 数据源 | 定价 | 关键限制 |
|------|---------|--------|------|---------|
| **Elicit** | 文献综述、结构化数据提取、表格对比、系统评价 | Semantic Scholar 1.38 亿篇 | $10-65/月 | **仅英文**；仅学术论文；提取精度不稳定 |
| **Consensus** | 快速 Yes/No 答案、SciScore 论文质量评分 | Semantic Scholar | 免费+付费 | **仅英文**；仅论文发现，深度分析弱 |
| **Undermind** (YC) | AI 引导式深度搜索、discovery curve 估算覆盖率 | 学术论文 | 免费+Pro | **仅英文**；搜索 8-10 分钟；不可复现检索策略 |
| **Scite** | 引用分析（支持/反对/提及）、Smart Citations | 学术论文 | 付费 | **仅英文**；仅引用关系，无综合分析 |
| **Research Rabbit** | 论文网络可视化、关联发现 | 学术论文 | 免费 | **仅英文**；仅发现阶段，无分析能力 |
| **SciSpace** | 文献综述、数据提取、论文起草 | 学术论文 | 付费 | **仅英文**；通用流程 |

**共性特征**：
- 全部仅支持英文
- 数据源集中在 Semantic Scholar / PubMed
- 核心价值是"帮你更快找到和读完论文"
- 无一嵌入特定研究方法论

### 第二梯队：通用 Deep Research

| 产品 | 核心功能 | 差异点 |
|------|---------|--------|
| **ChatGPT Deep Research** | 多步网页浏览 + 报告生成 | 通用能力最强，但用户反馈质量波动大（GPT-5 后退步的讨论） |
| **Claude Advanced Research** | 类似 ChatGPT Deep Research | 推理深度强，但无专用研究 UI |
| **Perplexity** | 搜索 + 综合 + 引用 | SimpleQA 93.9% 准确率；速度快但深度有限 |
| **Gemini Deep Research** | Google 生态整合 | 数据源优势，但研究流程通用 |

**共性特征**：
- 通用能力强，但"什么都能做 = 什么都不深"
- 无领域知识、无方法论、无过程评估
- 用户投诉集中在：幻觉、质量波动、用量限制、非英文支持差

### 第三梯队：开源研究 Agent

| 产品 | 核心功能 | 技术亮点 |
|------|---------|---------|
| **STORM (Stanford)** | 多视角提问 → 大纲 → 长文章生成 | 模拟"写手与专家对话"；基于 DSPy |
| **Co-STORM** | STORM + 人类参与 + 动态知识图谱 | **最接近综合集成研讨厅概念**；有 Moderator 角色引导讨论 |

**Co-STORM 值得重点关注**：
- 多角色协作：LLM 专家 + 主持人 + 人类用户
- 动态 mind map：层次化概念结构，降低认知负荷
- 人类可观察也可主动介入
- 但目标是生成 Wikipedia 文章，不是结构化研究

## 关键发现

### 1. 全赛道的"英文墙"

**所有主流学术研究工具均仅支持英文。** 中文学术内容（尤其是人文社科、中国特色领域如系统科学）完全不在服务范围内。这不是功能缺失，是数据源和架构决定的。

### 2. 无人做"方法论驱动的研究"

所有工具的研究流程都是同一个模式：**搜索 → 收集 → 摘要/对比**。

差异化在数据源和 UX，不在研究方法论。没有任何工具说"我用 X 方法论来分析问题"。研究方法论完全依赖用户自己掌握。

### 3. 评估层普遍缺失

- Elicit：结构化提取但不评估研究质量
- Undermind：有 discovery curve（搜索覆盖度估算），但不评估分析质量
- 通用 Deep Research：无任何质量度量
- **没有任何产品对"研究过程"本身做质量评估**

### 4. STORM/Co-STORM 是最接近的参照物

Co-STORM 的设计哲学（多角色 + 人类参与 + 动态知识结构）与你之前讨论的"综合集成研讨厅"理念高度吻合。但 Co-STORM：
- 目标是写文章，不是做研究
- 无评估回流
- 无领域方法论编码
- 是研究项目，不是产品

### 5. 用户痛点汇总

从各平台用户反馈中提取的高频痛点：

| 痛点 | 频率 | 当前解决方案 |
|------|------|------------|
| 非英文内容支持差 | 极高 | 无 |
| 幻觉/不准确 | 高 | 引用溯源（Elicit/Scite） |
| 浅层摘要，缺乏深度分析 | 高 | 无 |
| 无法复现研究过程 | 中 | 无 |
| 不替代批判性思考 | 中 | 无（被认为是AI的本质限制） |
| 领域专业性不足 | 中 | 无 |

## 对你项目的启示

### 可能的差异化定位

1. **中文学术领域**：这是整个赛道的真空地带。没有任何主流工具覆盖中文学术内容。
2. **方法论驱动**：所有现有工具都是"通用研究流程"。方法论编码是空白。
3. **研究过程评估**：无人做。你已经有 ResearchEvaluator。
4. **Co-STORM 式人机协作 + 方法论**：Co-STORM 有协作，你有方法论。二者结合是无人区。

### 需要进一步验证的

- 中文学术 RAG 的市场需求有多大？（数据源从哪来？用户画像？）
- "方法论驱动"对终端用户有没有感知价值？（用户能否理解并愿意为此付费？）
- Co-STORM 的技术路径可否借鉴/整合？
