目标：先修复最近一次失败 run 的阻塞点，再在预算内继续推进当前主路径的一小步。

执行要求：

- 必须先读取 `.agent/runtime/failed-run.log` 和 `.agent/runtime/failed-run.json`。
- 先判断失败属于哪一类：环境/网络瞬时问题、仓库状态问题、代码缺陷、verify 失败、越权改动。
- 优先修复确定性阻塞，不要顺手展开大重构。
- 如果失败只是瞬时外部波动，也要留下最小必要的守护或降噪改动，避免同类问题立刻复发。
- 修复后必须重新跑 verify。
- 只有 verify 通过，才允许调用 `agent-nightshift checkpoint`。
- 最终消息必须明确说明：根因、修复动作、验证结果、剩余风险、是否已经继续推进主任务。
