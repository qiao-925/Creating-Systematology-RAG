# Working Memory Boost

> 工作记忆持久化：跨 session 不丢失上下文

---

## 何时触发

**立即做**：用户说"存档"、"完成"、"好"

**自动做**：
- 改了 >3 个文件，或写了 >1000 tokens
- 出现明确决策（"选X"、"定了"）
- 解决阻塞

**不做**：
- 纯探索，无结论
- 仅改 typo/格式
- 测试失败
- 用户说"不用存档"

---

## 节点标记（轻量级）

> 比 CP 更细的粒度，记录关键思考节点，后期再降噪

### AI 自动标记（语义判断）

遇到以下场景，立即在 ongoing/*.md 追加「节点记录」：

| 信号类型 | 示例 | 记录内容 |
|----------|------|----------|
| **新发现/洞察** | "我突然想到..."、"关键洞察是..." | 时间 + 洞察摘要 + 来源位置 |
| **思路转折** | "不对，应该换个方向..."、"重新思考..." | 时间 + 转折原因 + 新思路 |
| **关键图/代码** | ASCII 图、Mermaid、代码块 >20 行 | 时间 + 图/代码摘要 + 对话位置 |
| **决策形成** | "那就定这个方案"、"优先级确定了" | 时间 + 决策内容 + 备选方案 |
| **待确认/回文** | "这个后面再验证"、"Phase 2 再优化" | 时间 + 待办项 + 预计阶段 |

### 用户显式标记

用户说以下关键词，立即记录前后上下文：
- "记下来"、"这是重点"、"mark一下"
- "看图"、"看这段代码"
- "这是关键转折点"

### 节点记录格式

在 ongoing/*.md 的当前 Phase 下追加：

```markdown
### 节点记录（2026-04-13）

| 时间 | 类型 | 内容 | 位置 |
|------|------|------|------|
| 04:30 | 图 | 五层运行架构草图（CLD→FCM→D2D） | [对话](link) |
| 04:35 | 决策 | 四图优先级待确认（业务/工程/流程/数据） | - |
| 04:40 | 洞察 | 上下文容易丢失，需加强记录粒度 | [对话](link) |
```

### 降噪原则

- **先记录，后整理**：宁可多记，不要漏记
- **每周回顾**：合并冗余节点，提炼为正式决策/调研
- **阶段归档**：Phase 结束时，将节点历史压缩为 Checkpoint

## 操作流程

### Step 0：读取看板

所有操作必须先读取 `.working-memory/board.md`，向用户展示：
- 进行中任务列表
- 最近归档（前 3 个）
- 最新数据分析和 aha-moment 链接

### Step 1：确认动作

| 触发词 | 动作 |
|--------|------|
| 存档、记录、同步 | `checkpoint` |
| 读档、恢复、继续 | `resume` |

默认 `checkpoint`。

### Step 2：完整性检查

**L1**：board 列的任务 ↔ ongoing/ 文件同步，缺失的尝试从 Issue 恢复
**L2**：本地文件 vs `.checkpoint` 对比 mtime，冲突时问用户
**L3**：对比 `HEAD` 与 `checkpoint.git_ref`，偏离时警告

### Step 3：执行动作

#### 存档（checkpoint）

流程：
1. 读取本地 `ongoing/任务名.md`
2. 更新 Checkpoint 状态表（用户或 AI 已更新）
3. 更新 `.checkpoint`：
   ```json
   {
     "issue": 15,
     "phase": "research",
     "completed": ["CP-18"],
     "next": "CP-19: 工程架构设计",
     "git_ref": "a1b2c3d",
     "timestamp": "2026-04-12T18:00:00Z"
   }
   ```
4. 同步 `board.md`：更新该任务的「状态」「下一动作」「更新」列
5. 写入远程 Issue（正文 + 评论）

远程层极简：
- **Issue 正文**（5 行内）：
  ```
  综合集成法架构洞察 - CP-18 完成
  Status: 🟡 research
  Next: 工程架构设计
  Local: .working-memory/ongoing/2026-04-10-02_...
  Git: a1b2c3d
  ```
- **评论**：仅记录关键节点（CP 完成、阻塞、决策）

#### 读档（resume）

**判断模式：**

| 条件 | 模式 | 操作 |
|------|------|------|
| 本地文件存在且 mtime > 远程 | 热恢复 | 读取本地，验证 L3 |
| 本地文件缺失或 mtime < 远程 | 温恢复 | 从 Issue 重建本地 |
| 用户显式指定 `#issue` | 强制重建 | 从指定 Issue 重建 |

**温恢复流程：**
1. `gh issue view <id> --comments`
2. 解析 Issue 正文 + 最新评论
3. 重建 `ongoing/任务名.md`（精简三区块格式）
4. 重建 `.checkpoint`
5. 更新 `board.md`（如该任务未列出则添加）
6. 输出恢复摘要

### Step 4：后置完整性检查（L3）

- 再次验证 Git 状态
- 对比本次操作前后的 `.checkpoint` 变化
- 确认远程同步成功（可选：抽查 Issue 正文）

## 文档格式规范

### board.md（全局看板）

```markdown
# 工作记忆看板

## 进行中

| 任务 | 阶段 | 状态 | 下一动作 | 更新 |
|------|------|------|----------|------|
| [#15] 综合集成法架构 | research | 🟡 | 工程架构设计 | 04-12 |

## 最近归档

| 任务 | 结果 | 日期 |
|------|------|------|
| [#14] 项目方向对齐 | ⬜ | - |

## 数据分析

- [Bug 健康度](./analysis/README.md)

## Aha Moments

- [模型选择](../aha-moments/2026-...md)
```

### ongoing/*.md（单个任务）

**最佳实践（双轨结构）：**

```markdown
# Issue #{n}: {任务标题}

> 一句话定位：{简洁描述}

## 1. Checkpoint 时间线（倒序）

| 日期 | 内容 |
|------|------|
| {MM-DD} | {动词开头叙事} |

---

## 3. 执行视图（决策 + 任务）

> 决策 = 执行指令（低可逆），任务 = 执行步骤（高可逆）。
> 状态：✅已定 / ⏳待定(有建议值) / 🔄可回访

### 📥 {模块名}

**决策**

| ID | 决策 | 指令 | 状态 | 理由 |
|----|------|------|------|------|
| D{n} | {问句} | {动词开头指令} | ✅ | {一句话} |

**任务**

| ID | 任务 | 状态 | 产出 |
|----|------|------|------|
| T{n} | {动词开头任务} | 📋 | {产出物} |

---

## 4. 阻塞与回文

| 项目 | 类型 | 说明 | 处理时机 |
|------|------|------|----------|

---

## 5. 调研索引

> 只索引，不重复内容。

| 目录 | 文档 | 关联决策 |
|------|------|----------|
```

**双轨原则：**
1. **Checkpoint（人读）**：时间叙事——"我怎么走到这里的"。动词开头，倒序，纯叙事
2. **执行视图（AI读）**：当前态参考——"每层选了什么、做什么"。按模块分组，决策问句+指令，任务动词开头
3. **两者不合并**：同一决策在两边出现是不同视角，Checkpoint记"什么时候定了什么"，执行视图记"当前决策指令+理由"
4. **无冗余**：Checkpoint不记决策ID/状态/理由（执行视图的事），执行视图不记日期（Checkpoint的事）

**语言规范：**
- **Checkpoint**：动词开头叙事（锁定/选定/发现/认清/完成/确认/梳理/画出/推导出/确立/收录）
- **执行视图-决策列**：问句（"何时停止检索"/"如何增强输入"/"用什么激活函数"）
- **执行视图-指令列**：动词开头指令（"用 Tanh"/"混合停止：硬限制保底 + 饱和度智能收敛"）
- **执行视图-任务列**：动词开头（"调研..."/"实现..."/"定义..."/"构建..."/"测试..."）

**图标系统：**
- ✅ 已完成/已定 | ⏳ 待定(有建议值) | 🔄 可回访
- 📋 待办 | 🔴 阻塞 | 📝 回文/优化

## 一行回执

**存档：**
```
已存档 #15 | research 🟡 | Next: 工程架构设计
本地: .working-memory/ongoing/...md ✅ | Git: a1b2c3d ✅
```

**读档（热）：**
```
已恢复 #15 | 来源:本地 | research 🟡 | Next: 工程架构设计
```

**读档（温）：**
```
已恢复 #15 | 来源:远程 | research 🟡 | Next: 工程架构设计
重建: .working-memory/ongoing/...md | Git: 一致
```

## Edge Cases

- `gh` 未登录 → 提示登录后停止
- board/文件不一致 → 自动同步
- 时间戳冲突 → 问用户选哪个
- Git 偏离 → 警告并显示差异
- Issue 正文过长 → 压缩为 5 行

## 复制到新项目

```bash
cp .agent/runtime/working-memory-boost.md /new-project/.agent/runtime/
cp -r .working-memory /new-project/
rm /new-project/.working-memory/ongoing/*.md
echo "!.agent/runtime/" >> /new-project/.gitignore
# 更新 AGENTS.md 指向 runtime/working-memory-boost.md
```
