# /auto-diagnose

## 1. Command Summary
- 目的：在排障或执行过程中快速收敛问题，记录尝试并判断是否需要升级。
- 触发时机：出现报错、阻塞或需要排查的异常；规则 `testing_and_diagnostics_guidelines.mdc` 提醒时。
- 关联规则：`testing_and_diagnostics_guidelines.mdc`
- 依赖工具：日志读取工具、配置/依赖检查脚本（视项目情况调用）。

## 2. 输入
- 必需：当前遇到的问题描述、已执行的操作（若有）。
- 可选：错误日志、相关文件路径、运行环境信息。

## 3. 执行步骤
1. 记录问题概述：现象、时间、影响范围。
2. 按规则执行最多三轮排查，每轮包含：
   - 收集最新日志或错误信息；
   - 检查配置、依赖、实现逻辑；
   - 记下本轮推断与操作结果。
3. 若三轮后仍未解决：
   - 总结已尝试路径与结果；
   - 列出可能的后续选项（至少两个）；
   - 判断是否触发升级条件并准备反馈。
4. 输出排障结论：解决方案或待升级信息。

## 4. 输出
- 排障记录（轮次、操作、结果）。
- 结论：已解决或需升级（含建议的下一步）。
- 相关日志/链接（如有）。
