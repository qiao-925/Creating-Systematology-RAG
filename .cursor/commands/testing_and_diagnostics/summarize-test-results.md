# /summarize-test-results

## 1. Command Summary
- 目的：整理单元测试执行信息，便于写入日志与向用户汇报。
- 触发时机：完成 `/run-unit-tests` 后；规则 `testing_and_diagnostics_guidelines.mdc` 提醒时。
- 关联规则：`testing_and_diagnostics_guidelines.mdc`
- 依赖工具：测试执行记录（log/stdout）、必要时的汇总脚本。

## 2. 输入
- 必需：测试命令、通过和失败结果、失败原因。
- 可选：覆盖率报告、额外日志链接。

## 3. 执行步骤
1. 汇总所有测试命令与状态（通过/失败/跳过）。
2. 对失败用例，记录错误摘要与计划的补救措施。
3. 形成结构化输出，供日志或交付说明直接引用。
4. 若存在未运行的必要测试，提示补测并列入待办。

## 4. 输出
- 测试执行汇总表。
- 失败用例及下一步行动。
- 对应引用路径或附件（如覆盖率报告）。
