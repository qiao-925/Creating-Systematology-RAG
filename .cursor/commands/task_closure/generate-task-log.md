# /generate-task-log

## 1. Command Summary
- 目的：在 `agent-task-log/` 生成符合规范的任务日志。
- 触发时机：确认进入收尾流程时；规则 `task_closure_guidelines.mdc` 提醒时。
- 关联规则：`task_closure_guidelines.mdc`
- 依赖工具：日期命令、编辑器或脚本（视实现）。

## 2. 输入
- 必需：任务类型（来自 `TASK_TYPES.md`）、任务名称、任务概述、关键行动、测试信息、交付结果。
- 可选：关联文件路径、遗留问题、后续计划。

## 3. 执行步骤
1. 确认任务类型与命名：`YYYY-MM-DD-N_【task_type】任务名称-文档类型.md`。
2. 创建文件并补充标题 `# YYYY-MM-DD 【task_type】任务名称-文档类型`。
3. 依据规则清单撰写内容，包括：
   - 任务概述与元信息；
   - 关键步骤与决策；
   - 实施方法、测试执行、结果与交付；
   - 引用或链接相关资料。
4. 复核命名、日期、引用格式，并在对话中汇报日志位置。

## 4. 输出
- 新增的任务日志文件路径。
- 简要摘要（可粘贴到对话中）。
