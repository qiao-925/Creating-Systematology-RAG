# /run-optimization-review

## 1. Command Summary
- 目的：对任务执行情况进行六维度复盘，识别亮点与改进项。
- 触发时机：任务日志生成后、执行收尾流程时；规则 `task_closure_guidelines.mdc` 提醒时。
- 关联规则：`task_closure_guidelines.mdc`
- 依赖工具：任务日志、相关代码/文档变更记录。

## 2. 输入
- 必需：任务范围、涉及模块或文件清单。
- 可选：性能数据、测试报告、用户反馈。

## 3. 执行步骤
1. 在日志中建立“任务优化分析”章节。
2. 针对六大维度逐项评估：
   - ✅ 至少一项做得好的点；
   - ⚠️ 总计不少于两项的改进建议（含影响、建议、优先级 🔴/🟡/🟢）。
3. 汇总优先级：立即处理 / 近期处理 / 长期规划。
4. 记录分析日期（`YYYY-MM-DD`），必要时附带后续行动提醒。

## 4. 输出
- 完整的优化分析段落，可直接写入任务日志。
- 建议与优先级列表。
