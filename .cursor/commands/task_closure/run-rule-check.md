# /run-rule-check

## 1. Command Summary
- 目的：核对本次任务匹配的规则执行情况，生成规则执行校验报告。
- 触发时机：任务收尾阶段，日志与优化分析完成后；规则 `task_closure_guidelines.mdc` 提醒时。
- 关联规则：`task_closure_guidelines.mdc`
- 依赖工具：规则清单（Always/Auto/Manual）、本次任务执行记录。

## 2. 输入
- 必需：任务名称、类型、执行中涉及的规则列表、各规则执行状态。
- 可选：补救措施说明、健康度评估依据。

## 3. 执行步骤
1. 收集当次任务匹配到的规则（按触发方式分类）。
2. 对每条规则标记状态：✅ 已执行、⚠️ 补救中、❌ 未执行（含原因）。
3. 统计执行率与补救率，给出规则系统健康度评估。
4. 在任务日志中追加“规则执行校验”章节，记录以上结论。

## 4. 输出
- 规则执行统计（列表或表格）。
- 健康度评估与后续补救计划。
