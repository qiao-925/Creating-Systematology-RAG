# /design-rule

## 1. Command Summary
- 目的：在新增或优化规则时提供结构模板与评审清单，确保输出符合规范。
- 触发时机：需要创建/调整规则文件、评审规则体系时；规则 `rule_authoring_guidelines.mdc` 提醒时。
- 关联规则：`rule_authoring_guidelines.mdc`
- 依赖资料：`TASK_TYPES.md`、既有规则案例、项目需求文档。

## 2. 输入
- 必需：规则适用场景、触发方式（Always/Auto/Manual）、核心约束。
- 可选：候选命令列表、参考资料链接。

## 3. 执行步骤
1. 获取现有规则模板，包括 Front Matter（`description`、`autoAttach`/`manualTrigger`、`globs` 等）。
2. 列出至少两种方案或结构，评估优劣并选择最合适的实现。
3. 使用检查清单确保：
   - 行数 ≤ 500，核心要求前 50 行。
   - 清单式或步骤式表达，避免冗长叙述。
   - 示例引用合理，引用路径使用反引号。
4. 如涉及命令，明确命令名称、触发条件，并在映射表中登记。
5. 准备评审要点：适用范围、执行清单、禁止事项、版本信息。

## 4. 输出
- 新规则或更新草稿。
- 方案比较记录与最终选择理由。
