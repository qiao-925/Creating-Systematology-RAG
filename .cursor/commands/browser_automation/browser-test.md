# /browser-test

## 1. Command Summary
- 目的：执行浏览器自动化测试，包括功能测试、视觉回归测试、响应式测试
- 触发时机：需要验证 UI 功能或进行回归测试时
- 关联规则：`browser_visual_editor_integration.mdc`
- 依赖工具：`@browser`

## 2. 输入
- 必需：测试类型（功能/视觉/响应式/全部）
- 可选：测试场景描述、截图保存路径

## 3. 执行步骤

### 3.1 功能测试
1. 确保应用已启动
2. 使用 `@browser` 打开应用
3. 执行测试场景：
   - 填写表单
   - 点击按钮
   - 验证交互响应
   - 检查控制台错误
   - 监控网络请求
4. 截图记录关键状态
5. 生成测试报告

### 3.2 视觉回归测试
1. 打开应用并截图记录当前状态
2. 与基准截图对比（如存在）
3. 识别视觉差异
4. 生成差异报告
5. 更新基准截图（如测试通过）

### 3.3 响应式测试
1. 在不同窗口尺寸下截图：
   - 桌面（1920x1080）
   - 平板（768x1024）
   - 手机（375x667）
2. 验证布局适配
3. 检查元素溢出或重叠
4. 生成响应式测试报告

## 4. 输出
- 测试执行摘要
- 截图保存路径
- 测试结果（通过/失败）
- 问题列表（如有）
- 改进建议

## 5. 示例
```
用户：/browser-test 功能测试 - 测试聊天输入和查询响应

执行：
1. 打开浏览器
2. 截图记录初始状态
3. 在聊天输入框输入测试查询
4. 点击发送按钮
5. 等待响应并截图
6. 检查控制台错误
7. 验证响应格式
8. 生成测试报告
```
