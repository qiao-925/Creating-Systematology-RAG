# /browser-a11y

## 1. Command Summary
- 目的：执行无障碍审计，检查并改进网页无障碍访问性
- 触发时机：需要验证或改进无障碍性时
- 关联规则：`browser_visual_editor_integration.mdc`
- 依赖工具：`@browser`

## 2. 输入
- 可选：检查范围（全部/特定页面）、WCAG 级别（A/AA/AAA）

## 3. 执行步骤
1. 确保应用已启动
2. 使用 `@browser` 打开应用
3. 执行无障碍检查：
   - **颜色对比度**：检查文本与背景对比度（WCAG AA/AAA）
   - **语义化 HTML**：验证标题层级、列表、表单结构
   - **ARIA 标签**：检查缺失或错误的 ARIA 属性
   - **键盘导航**：测试 Tab 键顺序、焦点管理
   - **Alt 文本**：检查图片是否有 alt 属性
   - **表单标签**：验证表单元素是否有关联标签
4. 生成问题列表和改进建议
5. 如需要，使用可视化编辑器修复问题

## 4. 输出
- 无障碍检查结果摘要
- 问题列表（按严重程度分类）
- 改进建议（含具体修复方案）
- 合规性评分（WCAG A/AA/AAA）

## 5. 示例
```
用户：/browser-a11y 检查主页和聊天界面的无障碍性

执行：
1. 打开浏览器
2. 检查颜色对比度
3. 验证键盘导航
4. 检查 ARIA 标签
5. 验证语义化 HTML
6. 生成改进建议报告
```
