# 文档驱动开发规范

> 在生成代码或修复 bug 时，优先查阅官方文档和示例，提升实现准确率。

---

## ⚠️ 核心强制要求

**在实现功能或修复 bug 前，优先查阅相关库/框架的官方文档。**

### 应该查阅文档的情况

- ✅ 使用第三方库/框架的新 API
- ✅ 遇到 API 调用错误（`AttributeError`、`TypeError`）
- ✅ 不确定正确的调用方式或参数
- ✅ 需要了解版本差异和兼容性

### 可以跳过文档的情况

- ⚠️ 使用项目内部已封装的 API
- ⚠️ 简单的语法错误或拼写错误

---

## AI Agent 行为要求

### 遇到 API 错误时

1. **立即停止猜测**：不要基于错误信息猜测正确的 API
2. **使用 web_search 工具**：搜索官方文档和 API 参考
3. **基于文档实现**：使用文档中的正确 API

### 实现新功能时

1. 先查阅相关文档
2. 确认 API 的正确用法
3. 参考官方示例代码
