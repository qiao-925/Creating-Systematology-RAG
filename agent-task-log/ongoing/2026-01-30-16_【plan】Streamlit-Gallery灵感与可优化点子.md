## 1. 背景与原则

### 1.1 思路转变

- 此前讨论：终端风 UI 改造、Streamlit 改造难度、主题仅 light/dark。
- **转变**：不硬改框架，而是在 **Streamlit 限制内** 找可落地的优化点子。
- **来源**：从 Streamlit 官方 Gallery 示例中提炼布局、交互、组件用法等灵感。

### 1.2 原则提醒

**官方文档 first**：官方文档与 Gallery 包含最新能力与最佳实践，优先从这里获取链接和灵感，再考虑社区或自定义方案。

## 2. 目标

- 收集并浏览 Streamlit 官方 Gallery 示例。
- 记录每个示例的 **可借鉴点**（布局、组件组合、交互模式、视觉层次等）。
- 筛选出 **与本项目 RAG/聊天场景相关**、且在 Streamlit 原生能力内可实现的优化点子。
- 产出：链接清单 + 简短灵感笔记（可配截图），供后续迭代 UI/UX 时参考。
