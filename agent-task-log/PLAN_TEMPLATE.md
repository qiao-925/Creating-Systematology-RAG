# 计划书模板

> 用于长程任务的规划与 Checkpoint 追踪。

---

## 使用说明

1. 复制本模板，创建新文件到 `ongoing/` 目录
2. 命名格式：`YYYY-MM-DD-N_【plan】任务名称-实施计划.md`
3. 填充各章节内容，重点是 **Checkpoint 状态表**
4. 执行过程中更新 CP 状态
5. 完成后通过 `/generate-task-log` 归档

---

## 状态标记说明

| 标记 | 含义 |
|------|------|
| ⬜ | 待开始 |
| 🔄 | 进行中 |
| ✅ | 已完成 |

---

---

## 最佳实践

1. **Checkpoint 粒度**：每个 CP 应该是 1-2 小时可完成的独立单元
2. **验收标准明确**：使用可勾选的 checklist，便于验收
3. **及时更新状态**：每完成一个 CP，立即更新状态表
4. **记录变更**：如需调整计划，在文档中说明变更原因
5. **保持精简**：计划书控制在 300 行以内，避免过度详细
