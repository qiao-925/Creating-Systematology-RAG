## 阅读路线图（3个阶段）

### 第一阶段：技术基础验证（必读，2篇）

**目标**：验证 LLM 能否自动生成 CLD，理解 FCM 多专家融合

#### 1. LLM 自动生成 CLD 可行性
- **论文**: arXiv 2503.21798 — "Leveraging LLMs for Automated CLD Generation" (ISDC 2024)
- **链接**: https://arxiv.org/abs/2503.21798
- **阅读重点**:
  - LLM 生成 CLD 的质量评估方法
  - Prompt 设计技巧
  - 简单结构 vs 复杂结构的性能差异
- **对项目的价值**: 决定是否可以用 LLM 自动构建 CLD，这是整个项目的技术基础

#### 2. FCM 多专家融合数学基础
- **论文**: arXiv 2111.12749 — "FCMpy: Constructing and Analysing FCMs in Python"
- **链接**: https://arxiv.org/abs/2111.12749
- **阅读重点**:
  - FCM 的数学定义（权重矩阵、迭代计算）
  - 多专家输入的模糊逻辑聚合方法
  - FCMpy 的 API 设计
- **对项目的价值**: 提供多 Agent CLD 融合的数学基础和工程实现参考

---

### 第二阶段：核心输出能力（必读，1篇）

**目标**：实现 D2D 杠杆点分析

#### 3. D2D 方法（CLD → 探索性仿真）
- **论文**: arXiv 2508.05659 — "Diagrams-to-Dynamics: Exploring CLD Leverage Points under Uncertainty" (2025)
- **链接**: https://arxiv.org/abs/2508.05659
- **阅读重点**:
  - CLD + 变量分类标签 → 自动生成 SD 模型
  - Monte Carlo 参数采样
  - 杠杆点排序算法
  - 不确定性估计
- **对项目的价值**: 项目核心输出方法，将 CLD 转为可推演的杠杆点分析

---

### 第三阶段：架构设计参考（建议读，2篇）

**目标**：设计 Conductor + Agent 架构和多 Agent 融合机制

#### 4. Pipeline Algebra 方法
- **论文**: MDPI Systems 13(9):784 — "LLM-Powered, Expert-Refined CLD via Pipeline Algebra" (2025)
- **链接**: https://www.mdpi.com/2079-8954/13/9/784
- **阅读重点**:
  - CLD 构建的流水线步骤分解
  - Human-in-the-loop 的验证机制
  - 符号验证方法
- **对项目的价值**: 契合 Conductor + Agent 架构，提供交互协议设计参考

#### 5. 多 Agent CLD 融合工程路径
- **论文**: PMC 2025 — "Integration of large-scale community-developed CLDs: NLP approach to merging factors based on semantic similarity"
- **链接**: 需查找 PMC 具体链接
- **阅读重点**:
  - Sentence Transformer 语义相似度归并
  - F1=0.68 的融合效果评估
  - 592 个节点的规模化处理
- **对项目的价值**: 多 Agent 各自构建 CLD 后的节点融合工程参照
