# 决策规则手册

## 总则

**所有决策必须先分类，再选择规则。没有万能规则，只有适配规则。**

---

## 规则一：Cynefin域判定（第一步）

### 判定标准（30秒内完成）

| 问题 | 是 → 进入该域 | 否 → 继续问 |
|------|--------------|------------|
| **因果关系已知？** | **Clear域** → 用规则2 | 问下一题 |
| **需要专家分析才能知？** | **Complicated域** → 用规则3 | 问下一题 |
| **事后才能知？** | **Complex域** → 用规则4 | 问下一题 |
| **需立即行动防崩？** | **Chaotic域** → 用规则5 | 默认Complex域 |

### 快速判定口诀

```
已知因果 → Clear（最佳实践）
分析可知 → Complicated（专家分析）
事后才知 → Complex（探测实验）
立即行动 → Chaotic（先干再说）
```

## 规则二：Clear域执行标准

### 适用场景
- 标准化流程（单次查询流水线）
- 重复性操作（数据提取、格式转换）
- 因果关系明确（输入X→输出Y确定）

### 执行规则

**Rule 2.1: 零决策原则**
```
Clear域中，AI只执行，不做选择。
所有决策点前移：由人在设计期预设，运行时固定执行。
```

**Rule 2.2: 三固定原则**
```
固定输入格式 → 不接受变体
固定处理流程 → 不允许分支
固定输出格式 → 不灵活适配
```

**Rule 2.3: 检查清单驱动**
```
每个Clear域任务必须有检查清单。
执行者（人或AI）逐条勾选，缺一项不可进入下一步。
```

### CLDFlow应用示例

| 组件 | Clear域判定 | 执行方式 |
|------|------------|----------|
| 文档解析 | 输入PDF→输出文本，因果明确 | 零决策，固定解析器 |
| 提取Prompt | 模板预设，运行时只填充 | 零决策，固定3个视角模板 |
| JSON输出 | Schema固定，不灵活适配 | 零决策，严格Schema校验 |
| 基础仿真 | Kosko算法，无参数调整 | 零决策，固定激活函数 |

## 规则三：Complicated域执行标准

### 适用场景
- 需要专家判断（阈值调整、算法选择）
- 多方案对比（框架评估、技术选型）
- 分析后可预测（因果关系存在但需挖掘）

### 执行规则

**Rule 3.1: 多框架并行原则**
```
必须同时探索至少3个不同框架/方案。
单一方案不可直接进入决策，必须有对比基准。
```

**Rule 3.2: 72小时冷却原则**
```
Complicated域决策必须在首次动议后等待72小时。
期间强制收集反对意见和替代方案。
紧急程度不足以跳过冷却期 = 实际不紧急。
```

**Rule 3.3: 专家验证原则**
```
每个Complicated决策必须通过至少1位外部专家验证。
专家定义：在该领域有3年以上实战经验者。
验证形式：书面反馈或结构化对话记录。
```

### CLDFlow应用示例

| 决策 | Complicated判定 | 执行要求 |
|------|----------------|----------|
| 归并阈值 | 0.7/0.8/0.85需分析选择 | 3方案对比+72小时冷却 |
| 冲突消解策略 | 投票/人工/延迟各有优劣 | 多框架评估+专家咨询 |
| 聚合算法 | 均值/加权/中位数适用场景不同 | 3方案对比+测试验证 |

## 规则四：Complex域执行标准

### 适用场景
- 系统演化（架构无法预设）
- 涌现行为（多Agent交互结果不可预测）
- 需求变化（用户需要随时间浮现）
- 技术迭代（底层技术快速变化）

### 执行规则

**Rule 4.1: 放弃预测原则**
```
Complex域禁止做长期详细计划。
计划范围不得超过6周，计划粒度不得细于周。
任何"6个月后我们将..."的表述视为违规。
```

**Rule 4.2: 探测优先原则**
```
任何假设必须经过小规模实验验证，才能扩大投入。
实验规模：投入不超过总预算的10%。
实验周期：不超过2周。
实验通过标准：明确定义，实验前书面确认。
```

**Rule 4.3: 接口隔离原则**
```
Complex域内部允许复杂性，但对外接口必须稳定且简单。
接口变更必须通过Complicated域评审（Rule 3）。
内部重构不得影响外部依赖。
```

**Rule 4.4: 信号响应原则**
```
建立早期信号识别机制，对变化快速响应。
信号定义：3个可观测指标，任一突破阈值即触发响应。
响应时限：信号出现后48小时内必须启动响应流程。
```

### CLDFlow应用示例

| 组件 | Complex判定 | 执行要求 |
|------|------------|----------|
| 多Agent协作 | 涌现行为不可预测 | 探测实验→小规模3Agent→观察→调整 |
| 架构演化 | 无法预设最终形态 | 6周滚动规划+接口隔离 |
| 技术选型 | Embedding/LLM快速迭代 | 抽象接口，允许切换实现 |
| 用户需求 | 政策分析需求随时间浮现 | 早期信号识别+快速响应 |

## 规则五：Chaotic域执行标准

### 适用场景
- 系统崩溃（服务不可用）
- 数据泄露（安全事件）
- 重大错误（输出误导性结论）

### 执行规则

**Rule 5.1: 先行动原则**
```
Chaotic域禁止过度分析，必须先采取行动稳定局面。
行动目标：阻止恶化，而非解决问题。
行动时限：识别后15分钟内必须行动。
```

**Rule 5.2: 分层响应原则**
```
第一层（0-15分钟）：自动止损（熔断、降级、关闭）
第二层（15分钟-2小时）：人工介入，稳定核心功能
第三层（2小时-24小时）：根因分析，制定修复计划
```

**Rule 5.3: 事后复盘原则**
```
Chaotic事件平息后24小时内必须完成复盘。
复盘产出：1个新增监控指标 + 1个新增应急预案条目。
```

## 规则六：决策分层原则

### 分层结构

```
战略层（月级）    ← 架构方向、里程碑
    ↓ 通过接口契约同步
战术层（周级）    ← 模块设计、依赖协调
    ↓ 通过API契约同步
反应层（秒级）    ← 代码执行、数据处理
```

### 执行规则

**Rule 6.1: 层内自治原则**
```
每层独立决策，不跨层实时协调。
战略层不干预战术层具体实现，只验收接口输出。
战术层不干预反应层具体代码，只验收API行为。
```

**Rule 6.2: 契约优先原则**
```
层间通过明确契约（接口/API/Schema）交互。
契约变更必须通过Complicated域评审（Rule 3）。
契约未定义区域 = 下层自治区域，上层不得干预。
```

## 规则七：资源分配原则（重心法则）

### 执行规则

**Rule 7.1: 80/20重心原则**
```
80%资源投入20%关键决策（Type 3）。
Type 3定义：框架选择、核心架构、战略方向。
其余80%决策用默认值快速推进。
```

**Rule 7.2: 三问筛选法**
```
每个声称的"关键决策"必须通过三问：
1. 这个决策现在不定，项目会立即停滞吗？
2. 这个决策错了，6个月后发现，代价是什么？
3. 这个决策对了，能带来10倍收益吗？
任一回答"是" = Type 3，其余 = 非关键。
```

## 规则八：人机分工原则

### 决策类型分工表

| 决策类型 | 定义 | 人负责 | AI负责 | 交互模式 |
|----------|------|--------|--------|----------|
| **Type 1** | 计算型，有明确最优解 | 设定目标 | 全部执行 | 人：验收结果 |
| **Type 2** | 分析型，多方案权衡 | 选择框架 | 生成选项 | 人：选方案，AI：准备材料 |
| **Type 3** | 战略型，定义问题本身 | 主导定义 | 挑战/发散 | 人：决策，AI：反对意见 |

### 执行规则

**Rule 8.1: 类型前置判定**
```
每个决策开始前必须明确类型（1/2/3）。
类型判定错误 = 分工错误 = 必然失控。
```

**Rule 8.2: 禁止降级原则**
```
Type 3决策禁止降级为Type 2或Type 1处理。
Type 1决策禁止升级占用Type 3资源。
```

## 规则冲突解决

### 优先级（高→低）

1. **规则一**（域判定）> 所有其他规则
2. **规则五**（Chaotic域）> 其他域规则（危机时覆盖）
3. **规则六**（分层）> 单域规则（层间冲突时）
4. **规则八**（人机分工）> 执行规则（分工不清时）

## 附录：快速查询卡

### 30秒启动检查

```
□ 这是什么域？（Clear/Complicated/Complex/Chaotic）
□ 这是什么类型？（Type 1/2/3）
□ 这是什么层？（战略/战术/反应）
□ 是否Type 3？是→触发80/20重心原则
```

### 决策前强制问

```
Clear域：检查清单全勾了吗？
Complicated域：72小时了吗？专家问了吗？
Complex域：探测实验做了吗？计划超6周了吗？
Chaotic域：15分钟内行动了吗？
```

### 冲突时查

```
规则冲突？→ 按优先级解决
域不明确？→ 默认Complex域（最保守）
类型不明确？→ 强制按Type 3处理（最保险）
```
