# 实用决策指南

## 核心观点

**大多数人决策太慢，因为把可逆决策当成不可逆决策来处理。**

---

## 一、唯一有用的分类：可逆 vs 不可逆

来源：Jeff Bezos, Amazon

### 可逆决策（Type 2）

**特征**：错了能改，成本可接受

**例子**：
- 选哪家供应商（可换）
- 用A方案还是B方案（可迭代）
- 招这个人（可解雇）

**怎么做**：
```
不用完美信息 → 快速决定 → 执行 → 观察 → 调整
```

**关键**：不需要大量分析，错了再改。

---

### 不可逆决策（Type 1）

**特征**：错了很难改，或改的成本极高

**例子**：
- 卖公司（卖了难买回）
- 核心架构选择（改=重构一切）
- 结婚（法律/情感成本高）

**怎么做**：
```
放慢 → 收集信息 → 深度分析 → 咨询他人 → 决定
```

**关键**：值得花时间，因为错了代价大。

---

### 10秒判定法

```
问：这个决策错了，我能承受代价吗？

能 → 可逆 → 快速做，不用想太多
不能 → 不可逆 → 停下来，认真想
```

**现实**：90%的决策都是可逆的，但你用处理不可逆的方式在处理它们。

---

## 二、对抗决策疲劳

来源：决策心理学研究

### 问题

人每天的决策精力有限。小事决策太多，大事没精力了。

**症状**：
- 下午很难做决定
- 纠结小事（吃什么、穿什么）
- 重要决策拖延

---

### 解法1：自动化低价值决策

**做法**：把重复决策变成固定规则

```
例子：
- 工作日午餐固定3家店轮换（不纠结）
- 衣服固定搭配（不纠结）
- 邮件固定时间回复（不随时看）
```

**效果**：省下的精力留给重要决策。

---

### 解法2：重要决策放在上午

**做法**：把不可逆决策安排在精力最好的时候

```
上午：做重要决策（可逆/不可逆分类、深度分析）
下午：做执行工作（不需要新决策）
晚上：不做任何重要决策
```

**科学依据**：研究显示法官上午批准 parole 的比例比下午高得多。

---

### 解法3：决策数量上限

**做法**：每天最多做X个重要决策

```
建议：
- 普通人：每天 ≤ 3 个重要决策
- 决策者：每天 ≤ 5 个重要决策

超过？→ 推迟到明天，或授权他人
```

---

## 三、满意化 vs 最大化

来源：Herbert Simon，诺贝尔奖得主

### 最大化者

**特征**：寻找"最好"的选择，穷尽所有选项

**代价**：
- 决策时间极长
- 选完还后悔（怀疑还有更好的）
- 焦虑

---

### 满意化者

**特征**：找到"足够好"的选择就停

**做法**：
```
1. 设定最低标准
2. 找到第一个满足标准的选项
3. 停止搜索
```

**例子**：
```
买房：
- 最大化者：看100套，比较所有因素，纠结3个月
- 满意化者：标准（3房/学区/预算内）→ 看5套 → 第3套满足 → 买
```

---

### 什么时候用什么？

| 情境 | 推荐 | 理由 |
|------|------|------|
| 可逆决策 | 满意化 | 错了能改，不需要最好 |
| 不可逆+资源充足 | 最大化 | 值得找最优 |
| 不可逆+资源有限 | 满意化 | 够好就行，时间也是成本 |
| 信息不足 | 满意化 | 无法最大化（不知道还有什么选项） |

## 四、常见陷阱和简单应对

### 陷阱1：确认偏误

**特征**：只看支持自己观点的信息

**简单应对**：
```
强制问："什么证据会让我改变想法？"

写不出来 → 说明你在确认偏误
写出来了 → 去找那个证据
```

---

### 陷阱2：沉没成本

**特征**：因为已经投入了，所以继续投入

**简单应对**：
```
问："如果我是今天刚开始，还会做这个决策吗？"

不会 → 停止，沉没成本不是成本
会 → 继续
```

---

### 陷阱3：锚定效应

**特征**：被第一个看到的信息锁定

**简单应对**：
```
看到第一个信息后，强制等24小时再决策。

或：问一个完全不了解情况的人，听他们的第一反应。
```

---

### 陷阱4：群体思维

**特征**：大家都同意，所以我也同意

**简单应对**：
```
指定一个人专门唱反调。

会议结束前，这个人必须提出至少1个反对意见。
```

---

### 陷阱5：过度自信

**特征**：高估自己的判断准确性

**简单应对**：
```
预测时，强制写置信度（50%/70%/90%）。

事后复盘，看70%预测有多少真的对了。

通常发现：实际准确率 < 预测置信度
```

## 五、实用技巧清单

### 技巧1：10-10-10规则

来源：Suzy Welch

**问**：
```
这个决策：
- 10分钟后感觉如何？
- 10个月后感觉如何？
- 10年后感觉如何？
```

**用**：短期情绪干扰时（冲动决策）

---

### 技巧2：预演失败

来源：Gary Klein

**做法**：
```
决策前，假装这个决策已经失败了。

写："这个决策失败了，原因是..."

列出3个最可能失败的原因。

然后：现在能做什么来避免它们？
```

**用**：重大不可逆决策前

---

### 技巧3：最小化后悔

来源：Jeff Bezos

**问**：
```
"80岁时回头看，我会后悔没做吗？"

会 → 做
不会 → 不做
```

**用**：人生重大选择（职业、城市、关系）

---

### 技巧4：专家电话

**做法**：
```
重要决策前，打1个电话给懂行的人。

不是问"我该怎么做"，而是问：
- "这个领域最常见的错误是什么？"
- "如果我是你弟弟，你会建议我注意什么？"

时间：15-30分钟
成本：一顿咖啡
收益：避免踩坑
```

---

### 技巧5：决策日志

**做法**：
```
每个重要决策，记录：
- 当时怎么想的（理由）
- 预测结果是什么
- 置信度多少

3个月后复盘：
- 实际结果 vs 预测
- 为什么有偏差
```

**效果**：提高自我认知，减少重复错误

---

## 六、什么时候不做决策

### 不做比做更好的情况

```
1. 信息严重不足 → 等待更多信息
2. 情绪高涨时 → 睡一晚再做
3. 别人能做得更好 → 授权
4. 不做也没后果 → 不做
5. 时间能自然解决 → 等待
```

---

## 七、快速参考卡

```
每天开始前：
□ 今天最多做___个重要决策（填数字，建议≤3）

遇到决策时：
□ 错了能改吗？ → 能=快速做，不能=停下来
□ 信息够吗？ → 不够=等待，够=继续
□ 情绪稳吗？ → 不稳=推迟，稳=继续

决策时：
□ 可逆 → 满意化（够好就行）
□ 不可逆 → 预演失败 + 专家电话

决策后：
□ 记录决策日志（重要决策）
□ 设置检查点（何时评估结果）
```

---

## 核心总结

1. **90%决策是可逆的** → 快速做，不用完美
2. **只有10%需要深度分析** → 认真做，值得花时间
3. **对抗决策疲劳** → 自动化小事，大事放上午
4. **满意化通常优于最大化** → 够好就行，错了再改
5. **简单技巧 > 复杂框架** → 10-10-10、预演失败、专家电话
