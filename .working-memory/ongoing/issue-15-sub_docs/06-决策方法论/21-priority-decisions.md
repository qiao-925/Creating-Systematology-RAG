# 高价值决策优先级清单

## 核心原则

**决策无限，但高价值决策只有 5-7 个**。优先定这些，其余用默认值或延迟决定。

---

## P0 - 阻塞级（不做无法启动）

| 决策 | 影响范围 | 选项 | 建议默认值 |
|------|----------|------|------------|
| **提取输出格式** | 整个因果层数据流 | JSON Schema vs 自由文本 | JSON Schema（强制结构化） |
| **节点归并相似度阈值** | 图结构准确性 | 0.7 / 0.8 / 0.85 | **0.8**（平衡精确与召回） |
| **冲突分歧度阈值** | 融合质量 vs 人工负担 | 0.4 / 0.5 / 0.6 | **0.5**（中等分歧聚合，高分歧人工） |

**为什么这 3 个是 P0**：
- 提取格式错了 → 下游全崩
- 归并阈值偏了 → 概念错配或冗余
- 冲突阈值松了 → 矛盾结论污染系统

---

## P1 - 架构级（影响系统行为）

| 决策 | 影响范围 | 选项 | 建议默认值 |
|------|----------|------|------------|
| **FCM 激活函数** | 仿真稳态结果 | Tanh vs Sigmoid | **Tanh**（对称，适合正负权重） |
| **聚合算法** | 专家意见融合方式 | 均值 / 加权 / 中位数 | **均值**（Phase 1 无历史权重） |
| **敏感性扰动幅度** | 杠杆点识别敏感度 | 10% / 20% / 50% | **10%**（保守，避免过度反应） |
| **停止条件策略** | 成本与质量权衡 | 硬限制 / 智能收敛 | **混合**（硬限制保底 + 新信息<2 收敛） |

**为什么这 4 个是 P1**：
- 激活函数改结果分布 → 政策建议方向可能变
- 聚合算法影响权重矩阵 → 仿真基础
- 扰动幅度大 → 杠杆点排序不稳定
- 停止条件松 → 成本爆炸或提前终止

---

## P2 - 优化级（可后期调整）

| 决策 | 影响 | 默认值 | 调整时机 |
|------|------|--------|----------|
| Embedding 模型 | 归并准确性 | `all-MiniLM-L6-v2` | 中文效果差时换 BGE |
| 语言权重映射表 | 数值化精度 | 论文标准 7 档 | 根据提取结果微调 |
| 最大迭代次数 | 仿真收敛 | 100 步 | 出现振荡时调整 |
| 成本上限 | 单次查询预算 | $5 | 实际运行后校准 |
| 重试次数 | 容错性 | 1 次 | 稳定性差时增加 |

---

## P3 - 细节级（现在不用管）

- 具体超时秒数（300s 够）
- 日志详细级别（INFO）
- 代码风格检查严格度（推荐但不阻塞）
- 测试覆盖率阈值（核心函数 60%）

---

## 决策顺序建议

```
Week 1: 定 P0（3个决策）→ 原型可跑
Week 2: 定 P1（4个决策）→ 系统稳定
Week 3+: 根据运行数据调 P2
```

---

## 快速决策模板

对每个 P0/P1 决策，只需回答：

1. **这个决策现在必须定吗？**（是/否）
2. **选 A 还是 B？**（二选一，不展开）
3. **可以后续改吗？**（是/否）

如果（3）是，先选默认值跑起来。
