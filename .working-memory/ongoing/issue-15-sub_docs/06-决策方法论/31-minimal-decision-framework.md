# 极简决策框架

## 核心原则

**决策框架本身不应成为负担。**

---

## 唯一决策点

```
这个决策错了，能改吗？成本多大？
                │
        ┌───────┴───────┐
       低成本可改        高成本不可逆
         │                │
      【默认做】        【停下来想】
      快速推进          用深度框架
```

**全部框架，仅此一问。**

---

## 两套模式

### 模式A：默认模式（90%决策）

**触发条件**：错了能改，成本可接受

**执行方式**：
```
选默认值 → 快速执行 → 观察结果 → 调整
```

**不要求**：
- 不用判定域类型
- 不用72小时冷却
- 不用写对比文档

**唯一要求**：
- 记录决策（1句话）
- 设置检查点（何时评估结果）

---

### 模式B：深度模式（10%决策）

**触发条件**：错了很难改，或改的成本极高

**执行方式**：
```
用完整框架（哲学+策略+适配）
```

**典型场景**：
- 架构核心选择（改=重构整个系统）
- 数据格式契约（改=下游全崩）
- 团队结构定义（改=人/流程全动）

---

## 如何判定"成本"

### 快速判定（10秒内）

```
改这个决策需要：

□ 重写代码 > 1周？          → 高成本
□ 影响 > 3个下游系统？      → 高成本  
□ 需要协调 > 5个人？         → 高成本
□ 用户可见且难解释？         → 高成本

以上任一勾选 → 模式B（深度）
全部不勾 → 模式A（默认）
```

## CLDFlow实战

### 立即用这套重新分类之前的12个决策

| 决策 | 错了能改？ | 成本 | 模式 |
|------|-----------|------|------|
| 提取输出格式 | ❌ 难改 | 下游全崩 | **B** |
| 归并阈值 | ✅ 可调 | 重新跑即可 | **A** |
| 冲突阈值 | ✅ 可调 | 重新配即可 | **A** |
| 激活函数 | ✅ 可调 | 重新仿真即可 | **A** |
| 聚合算法 | ✅ 可调 | 重新算即可 | **A** |
| 多Agent协作模式 | ❌ 难改 | 架构基础 | **B** |
| 人机介入点 | ✅ 可调 | 配置调整 | **A** |
| 停止条件 | ✅ 可调 | 改参数即可 | **A** |
| 成本上限 | ✅ 可调 | 改配置即可 | **A** |

**结论**：只有**2个决策**需要用深度框架，其余全部默认值快速过。

## 默认值清单（模式A用）

```
归并阈值：0.8
冲突阈值：0.5
激活函数：Tanh
聚合算法：均值
扰动幅度：10%
停止策略：硬限制（10轮/$5/5分钟）
重试次数：1
最大迭代：100
```

**原则**：这些值错了都能改，先跑起来看效果。

---

## 深度模式触发清单（模式B用）

```
□ 提取输出格式定义
□ 多Agent协作模式
□ 三层架构边界（CLD/FCM/D2D职责）
□ 核心数据契约（层间接口）
```

**原则**：这些错了很难改，值得停下来深度思考。

---

## 执行节奏

### 日常（90%时间）

```
遇到问题 → 问"错了能改吗？"
       ↓
    能改 → 选默认值 → 做 → 记录 → 继续
```

### 关键时刻（10%时间）

```
遇到关键决策 → 问"错了能改吗？"
         ↓
      难改 → 打开完整框架 → 深度分析 → 决策
```

---

## 为什么这套更好

| 过度设计版 | 极简版 |
|------------|--------|
| 每个决策都要分类 | 只有关键决策才分类 |
| 四层域×三层类型×三层级 = 36种情况 | 只有2种模式 |
| 决策前 burden 重 | 决策前 burden 轻 |
| 容易陷入"分析瘫痪" | 默认行动，例外才停 |

---

## 核心洞察

**好的决策框架 = 减少决策负担，而非增加。**

- 不是"每个决策都做对"
- 而是"关键决策做对，其他快速试错"

**可逆性 > 完美性**

在复杂系统中，快速试错比慢速规划更能找到正确路径。
