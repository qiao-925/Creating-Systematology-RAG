# CLDFlow决策框架适配层

## 作用

**将通用决策框架（27号文档）映射到CLDFlow项目的具体业务流程。**

---

## 域映射表

| 通用域 | CLDFlow情境 | 判定标准 |
|--------|------------|----------|
| **Clear** | 单次查询流水线 | 输入文档→输出报告，流程完全预设 |
| **Clear** | 数据格式转换 | JSON Schema固定，无变体 |
| **Complicated** | 阈值参数调整 | 需分析选择（0.7/0.8/0.85） |
| **Complicated** | 算法选型 | 多方案需对比测试 |
| **Complex** | 多Agent协作演化 | 涌现行为不可预测 |
| **Complex** | 系统架构长期演进 | 6个月后形态不确定 |
| **Chaotic** | 生产故障 | 服务不可用/数据泄露 |

## 类型映射

### Type 1（计算型，AI自治）

| 通用定义 | CLDFlow具体 | 执行方式 |
|----------|-------------|----------|
| 有明确最优解 | 文档解析格式转换 | 固定解析器，无分支 |
| 可自动化 | 标准JSON Schema验证 | 自动校验，失败即报错 |
| 快速通过 | Embedding相似度计算 | 固定模型+阈值，直接输出 |

**具体清单**：
- [ ] PDF解析为文本（固定库，无配置）
- [ ] JSON Schema验证（固定规则）
- [ ] 余弦相似度计算（固定算法）
- [ ] 基础Kosko仿真（固定激活函数）
- [ ] 日志格式化输出（固定模板）

### Type 2（分析型，人机协作）

| 通用定义 | CLDFlow具体 | 执行方式 |
|----------|-------------|----------|
| 多方案权衡 | 归并阈值选择 | AI：计算各阈值效果 → 人：选0.8 |
| 需专家判断 | 冲突消解策略 | AI：列出投票/人工/延迟 → 人：选人工兜底 |
| 对比评估 | 聚合算法选择 | AI：测试均值/加权/中位数 → 人：选均值 |

**具体清单**：
- [ ] 节点归并阈值（0.7/0.8/0.85对比）
- [ ] 冲突分歧度阈值（0.4/0.5/0.6对比）
- [ ] FCM激活函数（Tanh vs Sigmoid对比）
- [ ] 敏感性扰动幅度（10%/20%对比）
- [ ] 停止条件策略（纯硬限制 vs 混合）

**强制冷却期**：72小时（从首次动议到决策）

### Type 3（战略型，人主导）

| 通用定义 | CLDFlow具体 | 执行方式 |
|----------|-------------|----------|
| 框架选择 | 多Agent协作模式定义 | 人：定义协作框架 → AI：挑战漏洞 |
| 核心架构 | 三层边界划分（CLD/FCM/D2D） | 人：确定职责 → AI：验证接口可行性 |
| 战略方向 | 人机介入点设置 | 人：设定介入规则 → AI：模拟极端场景 |

**具体清单**：
- [ ] 提取输出格式定义（核心数据契约）
- [ ] 多Agent协作模式（并行/串行/混合）
- [ ] 人机介入点（何时强制人工审核）
- [ ] 系统演进策略（Phase 1→2→3路径）

**三问验证**（强制通过才进入Type 3流程）：
1. 此决策不定，项目立即停滞？→ **是**（如：格式不定无法开发下游）
2. 决策错误，6月后代价不可承受？→ **是**（如：格式错误需全系统重构）
3. 决策正确，带来10倍收益？→ **否**（格式正确是基线，非倍增）

**结论**：提取输出格式 = Type 3，需80%资源深度研讨

## 规则A定制（域适配）

### A.1 Clear域定制

**零决策清单**：
```
组件：CausalExtractor.extract()
├── 输入：document(str), perspective(enum)
├── 流程：固定模板 → LLM生成 → 固定解析
├── 输出：List[CausalLink]
└── 禁止：动态模板、运行时Prompt修改、输出格式变体

组件：NodeMerger.merge()
├── 输入：List[Node]
├── 流程：固定模型 → 固定算法 → 固定阈值
├── 输出：List[NodeCluster]
└── 禁止：模型切换、阈值调整、算法变体
```

### A.2 Complicated域定制

**72小时流程**：
```
Day 0：问题识别（如"归并阈值选多少"）
   ↓
Day 1：AI准备材料
       ├── 计算0.7/0.8/0.85在测试集的效果
       ├── 收集领域论文的常用阈值
       └── 列出各阈值的优缺点
   ↓
Day 2-3：人类决策
       ├── 审阅AI准备的材料
       ├── 咨询1位NLP/知识图谱领域专家
       ├── 记录反对意见（强制：至少1条）
       └── 做出决策并记录理由
   ↓
Day 4：更新文档，冻结配置
```

### A.3 Complex域定制

**6周滚动规划**：
```
当前周期：2024-04-15 至 2024-05-27（6周）

Week 1-2：探测实验
├── 多Agent协作：跑3个Agent并行提取
├── 观察：冲突频率、融合难度、输出质量
└── 假设：3视角足够覆盖主要维度

Week 3-4：响应调整
├── 若冲突率>30%：增加Conductor融合规则
├── 若输出质量<70%：调整提取Prompt
└── 更新假设，设计新实验

Week 5-6：固化/再探测
├── 若假设验证：固化到Clear域规则
├── 若假设失败：设计新实验继续探测
└── 规划下一个6周周期
```

**早期信号指标**：
```
Signal 1：单次查询成本
├── 阈值：>$5
└── 触发：切换到cheaper model或减少Agent数

Signal 2：冲突消解率
├── 阈值：人工介入>50%
└── 触发：调整冲突检测阈值或改进融合算法

Signal 3：用户满意度（内部测试）
├── 阈值：<70%认为"有帮助"
└── 触发：回溯到提取层改进Prompt质量
```

### A.4 Chaotic域定制

**15分钟响应**：
```
故障识别：生产环境异常
   ↓
T+0分钟：自动触发L1
├── API熔断（停止接收新请求）
├── 降级模式（返回缓存结果）
└── 告警通知（PagerDuty/钉钉）
   ↓
T+15分钟：人工触发L2
├── 核心团队介入
├── 确定故障范围（部分/全部）
└── 启动备用方案（如切换到备用模型）
   ↓
T+2小时：启动L3
├── 根因分析
├── 修复方案制定
└── 预计恢复时间通知
   ↓
T+24小时：复盘
├── 故障时间线记录
├── 新增监控指标（如缺失的告警）
├── 新增应急预案条目
└── 更新Chaotic响应手册
```

## 规则B定制（分层）

### 层间契约（CLDFlow特定）

**战略→战术契约**：
```
输入：Phase 1目标定义
├── 功能范围：单查询完整流水线
├── 质量约束：输出需人工审核通过率>80%
├── 成本约束：单次<$5
├── 时间约束：6周内MVP
└── 输出：战术层设计方案

验收标准：
□ 方案覆盖全部功能范围
□ 质量/成本/时间约束可度量
□ 层间接口Schema已定义
```

**战术→反应契约**：
```
输入：节点归并模块设计
├── 接口：merge(nodes: List[Node]) -> List[NodeCluster]
├── 约束：响应<100ms，准确率>85%
└── 输出：具体实现代码

验收标准：
□ 单元测试通过率100%
□ 接口契约不变更
□ 性能/准确率达标
```

## 规则D定制（人机分工）

### CLDFlow特定禁止行为

```
严格禁止：
□ 在单次查询流水线中，AI动态调整提取Prompt
□ 在节点归并时，AI根据输入动态选择阈值
□ 在冲突检测时，AI自主决定是否人工介入
□ 将Type 3决策（如格式定义）降级为Type 1快速执行
□ 人类在Clear域干预代码级实现细节
```

## 规则E定制（冲突解决）

### CLDFlow特定冲突场景

**场景1：域判定争议**
```
争议："多Agent协作"是Complicated还是Complex？
判定方A：可分析各协作模式优劣 → Complicated
判定方B：实际运行效果事前不可知 → Complex

解决方案：
1. 初始判定：Complex（保守原则）
2. 探测实验：跑3种协作模式各2周
3. 实验后：若效果可预测，降级为Complicated
```

**场景2：层级越界**
```
争议：战术层想修改战略层定义的接口Schema

解决方案：
1. 触发Complicated域流程（Rule A.2）
2. 72小时冷却+3方案对比+外部验证
3. 决策：若验证通过，更新Schema版本
4. 执行：战术层按新版本实现
```

## 快速启动卡（CLDFlow定制版）

### 30秒启动检查

```
□ 这是什么域？
   Clear：单次查询流水线/数据转换
   Complicated：阈值调整/算法选型
   Complex：多Agent协作/架构演化
   Chaotic：生产故障

□ 这是什么类型？
   Type 1：有明确最优解（AI自治）
   Type 2：多方案对比（72小时冷却）
   Type 3：框架选择（80%资源深度研讨）

□ 这是什么层？
   战略：里程碑/方向
   战术：模块设计
   反应：代码执行
```
