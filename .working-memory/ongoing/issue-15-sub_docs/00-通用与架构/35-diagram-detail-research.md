# 架构图承载细节的方案调研

## 问题定义

**目标**：用图承载更多决策细节，减少对文档的依赖。

**约束**：
- ASCII图（纯文本，版本控制友好）
- 单图不能无限膨胀（可读性下降）
- 需要在简洁和详细之间平衡

---

## 方案对比

| 方案 | 优点 | 缺点 | 适用场景 |
|------|------|------|----------|
| **分层图（C4模型）** | 逐层展开，每层聚焦 | 需要多个图文件 | 复杂系统架构 |
| **缩进层级** | 单文件，逻辑清晰 | 深层缩进导致行过长 | 决策层级不深 |
| **子图链接** | 主图简洁，细节分离 | 需要跳转阅读 | 决策点较多 |
| **可折叠标记** | 阅读时可手动折叠 | 纯文本无自动折叠 | IDE支持折叠 |
| **Mermaid迁移** | 可渲染，可交互 | 失去纯文本优势 | 需要可视化时 |

---

## 推荐方案：分层图（C4模型风格）

### 核心思想

```
不是一张大图包含所有，而是4张图逐层展开：

L1-上下文图：用户和系统的关系（极简）
L2-容器图：五层流水线和核心决策（当前主图）
L3-组件图：每层内部的模块和决策（详细）
L4-代码图：接口定义和数据结构（最详细）
```

### 针对CLDFlow的分层建议

```
📁 00-通用与架构/
├── 00-业务流程图-L1-上下文.md    # 极简：用户→CLDFlow→报告
├── 00-业务流程图-L2-容器.md      # 当前主图：五层+核心决策
├── 00-业务流程图-L3-组件-CLD.md  # CLD层详细：提取→归并→冲突→融合
├── 00-业务流程图-L3-组件-FCM.md  # FCM层详细：评级→聚合→仿真
├── 00-业务流程图-L3-组件-D2D.md  # D2D层详细：扰动→排序→输出
└── 00-业务流程图-L4-接口.md       # 数据结构+接口契约
```


### L4-接口图（数据结构）

```
CausalLink Schema (JSON)
├── source: string          # 原因变量名
├── target: string          # 结果变量名  
├── polarity: enum          # + / -
├── evidence: string        # 原文引用片段
├── confidence: float       # 提取置信度 (0-1)
├── agent_source: string     # 来源Agent (政策/经济/社会)
└── tier: int               # 来源层级 (1/2/3/4)

冲突检测分歧度计算
├── polarity_diff: bool     # 极性是否相反
├── strength_diff: float    # 强度差异 (0-1)
├── evidence_overlap: float # 证据重叠度
└── divergence_score: float # 综合分歧度 (0-1)
```

## 方案2：主图+可折叠区块（单文件）

### 核心思想

使用Markdown的<details>标签或可折叠注释，在单文件中实现"展开/收起"。

```markdown
# 业务流程图

## 主图（始终可见）
[简化版五层图]

## 输入层详情 <details>
[点击展开：详细决策I1/I2/I3]

## CLD层详情 <details>  
[点击展开：详细决策D1-D5]

## FCM层详情 <details>
[点击展开：权重聚合决策...]
```

### 优点
- 单文件管理
- 阅读时自主控制细节级别
- GitHub/GitLab原生支持<details>折叠

### 缺点
- 非纯ASCII，需要Markdown渲染
- 编辑时折叠展开不方便

## 针对CLDFlow的建议

### 当前痛点

- 已有12个调研文档，分散在各个子文件夹
- 决策点分布在不同文档中
- 需要统一视图

### 推荐方案：分层图（方案1）

**理由**：
1. CLDFlow架构天然分层（输入→CLD→FCM→D2D→输出）
2. 每层内部有独立决策，适合独立展开
3. 不同角色看不同层级：
   - 产品经理看L1/L2
   - 架构师看L2/L3
   - 开发看L3/L4

### 实施步骤

1. **保留当前主图**作为L2-容器图
2. **新建3个L3组件图**：CLD详细、FCM详细、D2D详细
3. **每个L3图包含该层全部决策**：
   - 决策编号
   - 决策内容
   - 默认值/策略
   - 来源文档链接（可选，非必须）
4. **L1上下文图**：用于对外汇报，极简
5. **L4接口图**：用于开发，数据结构+接口契约

## 快速对比

| 你现在的痛点 | 推荐方案 | 效果 |
|--------------|----------|------|
| 单图太复杂 | 分层图 | L2简洁，L3详细 |
| 不想用文档 | 把决策移入L3图 | 图即详细设计 |
| 需要跳转 | L3图内嵌极简链接 | 减少跳转次数 |
| 维护困难 | 4层独立维护 | 每层变化不影响其他 |
