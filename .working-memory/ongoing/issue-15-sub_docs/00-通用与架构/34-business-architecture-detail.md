# CLDFlow 业务架构详细说明

## 架构决策确认（4个）

| 决策 | 内容 | 影响 |
|------|------|------|
| **决策1** | JSON Schema输出格式 | 全链路数据契约 |
| **决策2** | 全自动协作模式 | 3Agent并行+自动融合 |
| **决策3** | 无人工介入点 | 全程内部自动处理 |
| **决策4** | 高分歧用裁判Agent | 元投票评审机制 |

---

## 五层架构职责边界

### 输入层

**职责**：接收并标准化用户输入

**输入**：
- 用户自然语言查询
- 来源材料（PDF/网页/数据库）

**输出**：
- 标准化Query对象
- 解析后的文档集合

**关键决策**：无（Clear域，固定流程）

---

### CLD层（因果结构提取）

**职责**：从文档中提取因果变量和关系，构建共享因果图

**子模块**：

| 子模块 | 职责 | 输出格式 |
|--------|------|----------|
| **提取Agent-政策** | 提取政府相关变量和因果链 | CausalLink[] |
| **提取Agent-经济** | 提取市场机制相关变量 | CausalLink[] |
| **提取Agent-社会** | 提取居民行为相关变量 | CausalLink[] |
| **节点归并器** | 语义相似度>0.8自动归并 | NodeCluster[] |
| **冲突检测器** | 计算分歧度，分类处理 | 低/中/高分歧标记 |
| **裁判Agent** | 高分歧（>0.5）元投票评审 | 融合后的CausalLink |
| **CLD构建器** | 组装共享因果图 | NetworkX DiGraph |

**全自动流程**：
```
3个Agent并行提取
    ↓
JSON Schema输出（CausalLink列表）
    ↓
节点归并（阈值0.8自动处理）
    ↓
冲突检测（计算分歧度）
    ├── 低分歧（<0.3）→ 加权平均自动聚合
    ├── 中分歧（0.3-0.5）→ 加权平均+置信度标记
    └── 高分歧（>0.5）→ 送裁判Agent评审
    ↓
共享CLD（NetworkX DiGraph）
```

**无人工介入点**：
- 节点归并争议：阈值0.8自动处理，不等人
- 冲突因果链：高分歧自动送裁判Agent，不等人
- 融合质量：自动标记置信度，下游自行判断

---

### FCM层（模糊认知图）

**职责**：将定性因果图转化为定量权重矩阵，进行场景仿真

**子模块**：

| 子模块 | 职责 | 输出 |
|--------|------|------|
| **权重聚合器** | 多Agent语言权重→数值权重 | 权重矩阵 W[n×n] |
| **仿真引擎** | Kosko迭代求稳态 | 稳态状态向量 |
| **场景对比器** | 基准场景 vs 干预场景 | 差异矩阵 |

**全自动流程**：
```
共享CLD输入
    ↓
每条边：3个Agent语言评级（-VH ~ +VH）
    ↓
均值聚合 → 数值权重（-1 ~ +1）
    ↓
Kosko仿真（Tanh激活，100轮或收敛）
    ├── 基准场景（当前状态）
    └── 干预场景（政策干预）
    ↓
输出：权重矩阵 + 场景对比结果
```

---

### D2D层（杠杆点分析）

**职责**：识别高影响力节点，排序政策干预优先级

**子模块**：

| 子模块 | 职责 | 输出 |
|--------|------|------|
| **敏感性分析器** | 单节点10%扰动，测系统响应 | 影响力分数 |
| **不确定性计算器** | 多视角权重分歧→置信度区间 | 置信度标记 |
| **杠杆点排序器** | 按影响力+置信度综合排序 | 杠杆点列表 |

**全自动流程**：
```
权重矩阵输入
    ↓
对每个节点：
    ├── 施加+10%扰动
    ├── 重新仿真
    └── 计算||新稳态-原稳态||
    ↓
按影响力排序
    ↓
叠加置信度标记（来自权重分歧）
    ↓
输出：杠杆点排序（高影响力+高/中/低置信度）
```

---

### 输出层

**职责**：结构化呈现分析结果

**输出内容**：
1. 因果结构图（CLD可视化）
2. 场景对比表（FCM仿真结果）
3. 杠杆点排序（影响力+置信度）
4. 证据追溯（来源Agent+原文引用）

## 层间接口契约

### 接口1：输入层 → CLD层

```yaml
输入: ParsedQuery
├── query_text: str          # 用户原始查询
├── documents: List[Document]  # 解析后的文档集合
└── context: Dict              # 附加上下文

输出: CausalLink[] (JSON Schema)
├── source: str          # 原因变量
├── target: str          # 结果变量
├── polarity: enum       # + / -
├── evidence: str        # 原文引用
└── confidence: float    # 提取置信度（Agent自评）
```

### 接口2：CLD层 → FCM层

```yaml
输入: SharedCLD (NetworkX DiGraph)
├── 节点列表（归并后的变量）
└── 边列表（融合的CausalLink）

输出: WeightedFCM
├── weight_matrix: float[n][n]  # 聚合后的权重
├── confidence_matrix: float[n][n]  # 聚合置信度
└── divergence_notes: List[str]  # 分歧标记（中分歧边）
```

### 接口3：FCM层 → D2D层

```yaml
输入: WeightedFCM
├── weight_matrix
├── baseline_state: float[n]      # 基准场景稳态
└── intervention_states: Dict[str, float[n]]  # 各干预场景稳态

输出: LeverageAnalysis
├── leverage_points: List[NodeImpact]
│   ├── node: str
│   ├── impact_score: float      # 影响力分数
│   ├── confidence: enum          # high/medium/low
│   └── affected_nodes: List[str] # 受影响最大的节点
└── uncertainty_ranges: Dict[str, Tuple[float, float]]
```

### 接口4：D2D层 → 输出层

```yaml
输入: LeverageAnalysis + SharedCLD + FCMResults

输出: StructuredReport
├── cld_visualization: Graph Viz
├── scenario_comparison: Table
├── leverage_ranking: List[Recommendation]
└── evidence_tracing: Dict[str, Citation]
```

## 数据流全景

```
用户查询 + 文档
    ↓
[输入层] 标准化
    ↓
Query + Documents
    ↓
[CLD层] 并行提取 → 自动融合
    ├── 政策Agent ─┐
    ├── 经济Agent ─┼→ Conductor融合引擎（全自动）
    └── 社会Agent ─┘
    ↓
SharedCLD (DiGraph)
    ↓
[FCM层] 权重赋值 + 仿真
    ↓
WeightMatrix + ScenarioResults
    ↓
[D2D层] 敏感性分析
    ↓
LeverageRanking
    ↓
[输出层] 结构化报告
    ↓
用户
```

## 全自动处理点清单

| 处理点 | 旧设计（有人工） | 新设计（全自动） | 实现方式 |
|--------|-----------------|-----------------|----------|
| 节点归并争议 | 人工介入决定 | 阈值0.8自动处理 | 余弦相似度>0.8自动归并 |
| 冲突因果链 | 人工审核 | 裁判Agent评审 | 高分歧（>0.5）送元投票 |
| 融合质量判断 | 人判断是否可用 | 置信度标记输出 | 低/中/高分歧自动标记 |
| 杠杆点审核 | 人审核推荐 | 全自动排序 | 影响力+置信度综合排序 |

---

## 关键阈值汇总

| 阈值 | 值 | 用途 | 可调？ |
|------|-----|------|--------|
| 节点归并相似度 | 0.8 | 自动归并决策 | 是（Type 2） |
| 低分歧 | <0.3 | 自动聚合+高置信度 | 否（策略固定） |
| 中分歧 | 0.3-0.5 | 自动聚合+中置信度标记 | 否（策略固定） |
| 高分歧 | >0.5 | 送裁判Agent | 否（策略固定） |
| 敏感性扰动 | 10% | 杠杆点识别 | 是（Type 2） |
| FCM收敛 | 1e-4 | 稳态判断 | 是（Type 2） |
| 最大迭代 | 100 | 仿真上限 | 是（Type 2） |
